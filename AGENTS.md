# CarryMate Agent Guide

## 프로젝트 목적
- CarryMate는 대학생 팀 프로젝트 협업을 홈, 업무, 일정, 파일 4개 탭으로 정리하는 모바일 웹앱 프로토타입이다.
- MVP 목표는 더미 데이터와 React State만으로 발표에서 바로 시연 가능한 협업 흐름을 만드는 것이다.

## 코드 스타일
- TypeScript와 함수형 React 컴포넌트를 사용한다.
- 과도한 추상화보다 읽기 쉬운 상태 이름과 명확한 props를 우선한다.
- 데모용 로직은 단순하고 예측 가능하게 유지한다.

## UI 원칙
- 모바일 퍼스트만 고려한다.
- 화이트 배경, 넉넉한 여백, 둥근 카드, 부드러운 그림자를 유지한다.
- 브랜드 블루 `#1e70e6`는 핵심 액션과 강조 요소에만 사용한다.
- 신입생도 바로 이해할 수 있는 쉬운 문구를 사용한다.

## 기존 파일 보호 원칙
- 루트 `index.html`은 명시적 요청 전까지 삭제하거나 덮어쓰지 않는다.
- 새 앱 구조는 안전하게 추가하고, 기존 파일은 보존한다.
- 구조 변경이 필요하면 먼저 이유와 영향을 설명한다.

## MVP 우선 개발 원칙
- 로그인, API, DB, 서버 연동 없이 React State와 더미 데이터만 사용한다.
- 업무 추가, 상태 변경, 일정 확정, 파일 업로드, 최종본 지정 같은 발표용 상호작용을 먼저 완성한다.
- 기능 수를 늘리기보다 4개 탭의 연결감과 즉시 반응 UX를 우선한다.
